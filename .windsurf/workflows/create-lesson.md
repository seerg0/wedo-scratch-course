# Создание нового урока

Этот workflow помогает создать новый урок для курса программирования детей на WeDo + Scratch.

## Шаги для создания нового урока

### 1. Определите основную информацию
- Номер урока
- Тип урока (WeDo или Scratch или WeDo+Scratch)
- Название урока
- Основная концепция для изучения

**Важно:** Учитывайте формат курса из файла `0_формат_курса.md`:
- Возраст: 6–9 лет
- Продолжительность: 1 час (55 минут активной работы)
- Чередование: WeDo → Scratch → WeDo → Scratch

### 2. Скопируйте шаблон
Скопируйте содержимое файла `.windsurf/rules/lesson_template.md` в новый файл с именем:
- `[Номер]_[название_урока]_[тип].md` (например: `1_сигнализация_wedo.md`)

**⚠️ Важно:** Не удаляйте блок `---` в начале файла (YAML frontmatter) — он необходим для корректной работы Jekyll на GitHub Pages.

### 3. Заполните плейсхолдеры
Замените все плейсхолдеры в квадратных скобках `[...]` конкретными данными:

**Основные плейсхолдеры:**
- `[Номер]` - номер урока
- `[Тип]` - WeDo или Scratch
- `[Название урока]` - краткое название
- `[Общее время]` - обычно 55 минут
- `[Ключевая концепция]` - основная идея урока
- `[Что делаем]` - конкретные действия на каждом этапе

### 4. Адаптируйте под тип урока

**Для уроков WeDo:**
- Этап 2: Сборка модели с датчиками/моторами
- Компоненты: датчики, моторы, блоки WeDo
- Роли: Сборщик, Контролёр, Тестировщик

**Для уроков Scratch:**
- Этап 2: Создание спрайтов и фонов
- Компоненты: спрайты, сцены, блоки Scratch
- Роли: Художник, Программист, Тестировщик

### 5. Проверьте структуру
Убедитесь, что все разделы заполнены:
- ✅ Краткий план урока
- ✅ Цель и концепция
- ✅ Задача урока
- ✅ Детальный план с временными рамками
- ✅ Описание каждого этапа
- ✅ Советы для детей
- ✅ Критерии оценки

### 6. Обновите содержание сайта
После создания нового урока обновите `index.html`:

1. **Добавьте ссылку на новый урок** в секцию навигации:
   ```html
   <li><a href="[Номер]_[название]_[тип].md">[Номер]. [Название] ([Тип])</a> - [краткое описание]</li>
   ```

2. **Обновите общее количество уроков** если нужно

3. **Проверьте порядок уроков** - должен быть логичным

### 7. Сохраните файл
Сохраните файл в корневой папке проекта рядом с другими уроками.

### 8. Загрузите изменения
```bash
git add .
git commit -m "add: урок [Номер] - [Название]"
git push origin main
```

## Пример использования

1. Создать файл `2_сигнализация_scratch.md`
2. Скопировать в него шаблон
3. Заполнить:
   - `[Номер]` → `2`
   - `[Тип]` → `Scratch`
   - `[Название урока]` → `сигнализация`
   - `[Ключевая концепция]` → `Перенос логики из LEGO в игру`
   - И так далее...

## Результат

Готовый структурированный урок, который соответствует формату курса и готов к использованию.
